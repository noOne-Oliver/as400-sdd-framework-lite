## 5. 程序设计 ⚠️ 结构决定生成质量

### 5.1 程序清单

| 程序名 | 类型 | 说明 |
|--------|------|------|
| PGM001 | RPGLE | 主程序 |
| SRP001 | *SRVPGM | 公共服务 |

### 5.2 程序结构 ⚠️ 必填

```markdown
## 程序：PGM001

### 结构（用文字描述执行顺序）

1. **初始化**：打开文件，读取参数
2. **主处理**：读取记录 → 校验 → 处理 → 更新
3. **结束**：关闭文件，返回状态

### 关键逻辑位置

- 主循环：第 20-50 行
- 校验逻辑：第 25-30 行
- 更新逻辑：第 35-45 行

### 调用关系

```
PGMMAIN → PGM001 → ORDHDR（读）
                  → ORDLINE（读/写）
                  → MSGCTL（调用）
```

### 入口参数

| 参数 | 类型 | 方向 | 说明 |
|------|------|------|------|
| p_OrderNo | CHAR(10) | IN | 订单号 |
| p_Status | CHAR(1) | OUT | A=成功 |

```

## 6. Copilot 生成规则 ⚠️ 专门为 AI 写的说明

### 6.1 代码格式要求

```markdown
- ✅ 使用 RPGLE / FREE 格式
- ✅ 使用 DCL-PRC / DCL-PI 定义过程接口
- ✅ 使用 EXTNAME / LIKEDS 引用文件
- ❌ 禁止 *ENTRY PLIST（旧式）
- ❌ 禁止 C-spec 语法
```

### 6.2 文件引用规则

```markdown
- ✅ 使用 EXTNAME('文件名' : *OMITTED) 带字段列表
- ✅ 引用字段用 "FLDNAME" 双引号
- ✅ 字符处理用 %TRIM / %TRIMR
- ❌ 不要省略字段列表（会导致幻觉）
```

### 6.3 错误处理规则

```markdown
- ✅ 操作前判断文件状态 INFSPC
- ✅ 记录错误到 ERRLOG 或 job log
- ✅ 使用 MONITOR / ON-ERROR 捕获异常
- ❌ 不要省略 ELSE 分支
```

### 6.4 禁止代码模式 ⚠️ 重要

```markdown
// ❌ 禁止：假设字段存在
CHAIN KEYFIELD FILEA
IF %FOUND
  // 假设 FLDA 存在 - 不一定！
END

// ✅ 正确：明确处理不存在的情况
CHAIN KEYFIELD FILEA
IF %FOUND
  // 处理存在的情况
ELSE
  // 处理不存在的情况
  EXCEPT ERRLOG
ENDIF
```
