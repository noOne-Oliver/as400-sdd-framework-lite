# 团队静态检查规则

> 本文件用于存放团队内部的静态检查规则。
> Team Leader 负责维护此文件，Review 时按此清单检查。

---

## 使用说明

1. 在下方「检查规则」章节中填写团队的静态检查项
2. 格式：分为「必须修复」和「建议修复」两类
3. 提交 PR 时，Reviewer 会对照此清单检查

---

## 检查规则

### 命名规范

#### 程序命名
| 类型 | 格式 | 示例 | 必须修复 |
|------|------|------|----------|
| 主程序 | PGM + 功能缩写 | PGMORDH, PGMCUST | [ ] |
| 服务程序 | SRV + 功能缩写 | SRVORD, SRVCUST | [ ] |
| 子程序 | S_ + 功能缩写 | S_GETORD, S_VALDT | [ ] |
| 报表程序 | RPT + 序号 | RPT001, RPT002 | [ ] |
| 批处理程序 | BAT + 功能缩写 | BATORD, BATINV | [ ] |

#### 文件命名
| 类型 | 格式 | 示例 | 必须修复 |
|------|------|------|----------|
| 物理文件 | [前缀]F + 名称 | ORDHDRF, CUSTF | [ ] |
| 逻辑文件 | [前缀]L + 名称 | ORDHDRL1, ORDHDRL2 | [ ] |
| 数据结构 | [前缀]D + 名称 | ORDHDRD, CUSTD | [ ] |

#### 字段命名
| 类型 | 格式 | 示例 | 必须修复 |
|------|------|------|----------|
| 输入参数 | P_ + 名称 | P_ORDNO, P_CUST | [ ] |
| 工作字段 | W_ + 名称 | W_COUNTER, W_FLAG | [ ] |
| 指标字段 | Z_ + 名称 | Z_TOTAL, Z_AMT | [ ] |

---

### 代码格式

| 检查项 | 规范 | 必须修复 |
|--------|------|----------|
| 缩进 | 2 空格 | [ ] |
| 注释风格 | // 注释（单行）/* 注释（多行）*/ | [ ] |
| END 关键字 | IF/DO/FOR 等必须对应 END | [ ] |
| 大小写 | 关键字大写（IF, DCL, etc） | [ ] |
| 字符串引号 | 单引号 'text' | [ ] |

---

### RPG 特定规范

| 检查项 | 规范 | 必须修复 |
|--------|------|----------|
| 代码格式 | 必须使用 FREE 格式 | [ ] |
| 过程定义 | 必须使用 DCL-PROC / DCL-PI | [ ] |
| 文件引用 | 推荐使用 EXTNAME | [ ] |
| 错误处理 | 必须有 INFSPC 或 MONITOR | [ ] |
| 字符串比较 | 必须使用 %TRIM | [ ] |
| 日期字段 | 必须是 8 位数值 (YYYYMMDD) | [ ] |

---

### 安全检查

| 检查项 | 规范 | 必须修复 |
|--------|------|----------|
| 硬编码密码 | 禁止在代码中出现密码 | [ ] |
| 硬编码密钥 | 禁止在代码中出现 API Key | [ ] |
| SQL 注入 | 使用参数化查询 | [ ] |
| 敏感日志 | 禁止记录完整卡号等敏感信息 | [ ] |

---

### 性能检查

| 检查项 | 规范 | 必须修复 |
|--------|------|----------|
| 循环内查询 | 禁止在循环内 CHAIN 文件 | [ ] |
| 大数据量 | 超过 10000 条必须分批 | [ ] |
| 文件打开 | 必须在使用前打开，使用后关闭 | [ ] |
| COMMIT 频率 | 大数据量操作每 1000 条 COMMIT 一次 | [ ] |

## 可自动修复项（Fix Format）

以下问题 Copilot 可自动修复，Review 时发现可直接生成修复建议：

| 检查项 | 修复方式 | 说明 |
|--------|----------|------|
| 缩进不一致 | 重新格式化 | 将缩进统一为 2 空格 |
| 大小写不一致 | 重新格式化 | 关键字大写，标识符保持 |
| 缺少 ENDIF/ENDDO | 补充结构 | 根据缩进推断并补充 |
| 空格多余/缺失 | 重新格式化 | 统一空格风格 |
| 字符串引号不一致 | 统一为单引号 | 'text' |

## 审查规则

### 严重度定义

| 严重度 | 说明 | 处理方式 |
|--------|------|----------|
| **致命** | 编译失败、安全漏洞 | 必须修复，阻塞合并 |
| **高** | 业务逻辑错误、运行时错误 | 必须修复 |
| **中** | 规范不符、潜在风险 | 建议修复 |
| **低** | 代码风格、可读性 | 可忽略 |

### 合并条件

- [ ] 无「致命」和「高」级别问题
- [ ] 至少 1 人 Approve
- [ ] 所有静态检查已通过
