# Prompt: 代码 Review 辅助

## 任务
Review 以下 RPGLE 代码，找出潜在问题。

## 代码
```rpgle
[粘贴代码]
```

## Spec 要点
- 涉及文件：[文件名]
- 业务规则：[关键规则描述]
- 校验要求：[字段校验规则]
- 事务要求：[是否需要 COMMIT/ROLLBACK]

## Review 要求

### 1. 正确性
- [ ] 字段引用是否与文件定义一致？
- [ ] 逻辑判断是否符合业务规则？
- [ ] 异常处理是否完整？
- [ ] 所有 IF 都有 ELSE 分支？

### 2. 安全性
- [ ] 是否有硬编码敏感信息？
- [ ] 是否有 SQL 注入风险（如果用嵌入式 SQL）？
- [ ] 权限检查是否到位？

### 3. 性能
- [ ] 文件操作是否使用了最优方式（CHAIN vs READE）？
- [ ] 是否有不必要的循环？
- [ ] 事务控制是否合理（避免长事务）？

### 4. 代码质量
- [ ] 命名是否清晰？
- [ ] 是否有必要的注释？
- [ ] 是否遵循团队代码规范（FREE 格式）？

## Copilot 幻觉专项检查 ⚠️
- [ ] 所有 CHAIN/READE 的键值字段在文件中存在吗？
- [ ] 所有引用的字段名与 Spec 中一致吗？
- [ ] 有没有看起来合理但实际不存在的字段？
